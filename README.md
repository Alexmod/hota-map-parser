# Heroes 3 / HotA Map Analyzer

Небольшой Python-анализатор карт Heroes of Might and Magic III: Horn of the Abyss в формате `.h3m`.

Скрипт читает gzip-сжатые карты, достает базовую информацию, объектный слой и помогает отвечать на практические вопросы: где лежит артефакт, где взять заклинание, где находится герой, какие есть тюрьмы и Hero Camps.

## Возможности

- Сводка по карте: название, размер, подземелье, режим чтения, количество объектов.
- Поиск источников заклинаний:
  - святилища;
  - свитки;
  - scholars;
  - Хижины Провидца;
  - события;
  - ящики Пандоры;
  - принудительные заклинания в гильдиях магов городов.
- Поиск источников артефактов:
  - артефакты на карте;
  - события и ящики Пандоры;
  - награды Хижин Провидца;
  - награды монстров и банков существ, если карта разобрана полным режимом;
  - артефакты у героев.
- Поиск героев на карте, в тюрьмах и Hero Camps.
- Отдельная классификация Prison и Hero Camp для объектов класса 62.
- Fallback-режимы для новых или нестандартных HotA-карт, где полный последовательный разбор пока невозможен.

Координаты выводятся в нулевой системе координат H3M/парсера: `(x, y, z)`.

## Требования

- Python 3.10+.
- Внешние зависимости не нужны.

## Быстрый старт

```powershell
python .\analyze_h3m.py "<map.h3m>" summary
```

Примеры запросов:

```powershell
python .\analyze_h3m.py "<map.h3m>" summary
python .\analyze_h3m.py "<map.h3m>" spell "Town Portal"
python .\analyze_h3m.py "<map.h3m>" artifact "Golden Bow"
python .\analyze_h3m.py "<map.h3m>" hero "Gelu"
python .\analyze_h3m.py "<map.h3m>" prisons
python .\analyze_h3m.py "<map.h3m>" camps
python .\analyze_h3m.py "<map.h3m>" debug
```

Если команда не указана, используется `summary`.

## Команды

### `summary`

Печатает короткую сводку по карте.

```powershell
python .\analyze_h3m.py "<map.h3m>" summary
```

### `spell`

Ищет известные источники заклинания.

```powershell
python .\analyze_h3m.py "<map.h3m>" spell "Town Portal"
```

### `artifact`

Ищет известные источники артефакта.

```powershell
python .\analyze_h3m.py "<map.h3m>" artifact "Golden Bow"
```

Если артефакт находится у героя на карте, источник будет показан как `defeat-hero`: его можно получить, победив этого героя.

### `hero`

Ищет героя среди объектов карты.

```powershell
python .\analyze_h3m.py "<map.h3m>" hero "Gelu"
```

### `prisons` и `camps`

Показывают найденные тюрьмы и Hero Camps.

```powershell
python .\analyze_h3m.py "<map.h3m>" prisons
python .\analyze_h3m.py "<map.h3m>" camps
```

### `debug`

Печатает отладочную JSON-сводку: режим парсера, ошибки fallback-режима, количество шаблонов/объектов и самые частые классы объектов.

```powershell
python .\analyze_h3m.py "<map.h3m>" debug
```

## Ограничения

- Основной фокус - HotA `.h3m`. Классические RoE/AB/SoD-карты не обязаны разбираться.
- На некоторых новых или нестандартных HotA-картах анализатор переходит в fallback-режим сканирования объектов. В этом режиме надежны не все типы источников: размещенные объекты, свитки, святилища, события, ящики Пандоры, тюрьмы, Hero Camps и экипированные артефакты героев обычно читаются лучше, чем сложные вложенные награды.
- Маршруты через монолиты пока не являются отдельной CLI-командой основного анализатора.

## Практические правила интерпретации

- Для игровых ответов предпочитай `active=(x, y, z)`, а не `pos=(x, y, z)`.
- Монолит сам по себе не дает заклинание или артефакт; он может только открывать путь к области, где находится источник.
- Если результат выглядит на одну клетку смещенным относительно редактора, проверь, использует ли редактор нулевую или единичную систему координат.
