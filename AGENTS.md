# Инструкции проекта

Эта рабочая папка предназначена для анализа карт Heroes of Might and Magic III / Horn of the Abyss в формате `.h3m`.

Когда пользователь задает вопросы о карте Heroes 3 HotA из этого репозитория, используй скрипты, которые лежат прямо в рабочей папке. Инструкции должны оставаться переносимыми: не ссылайся на локальные пути Codex или файлы за пределами репозитория.

Структура:

- Основной анализатор: `.\analyze_h3m.py`.
- Документация проекта: `.\README.md`.
- Инструкции для агента: `.\AGENTS.md`.
- Если вопрос по карте выявляет отсутствующую возможность парсера или неверный результат, свободно правь `.\analyze_h3m.py`, затем проверяй исправление реальными командами по картам.

Для инспекции карт предпочитай скрипт из рабочей папки:

```powershell
python .\analyze_h3m.py "<map.h3m>" summary
python .\analyze_h3m.py "<map.h3m>" spell "Town Portal"
python .\analyze_h3m.py "<map.h3m>" artifact "Golden Bow"
python .\analyze_h3m.py "<map.h3m>" hero "Gelu"
python .\analyze_h3m.py "<map.h3m>" prisons
python .\analyze_h3m.py "<map.h3m>" camps
python .\analyze_h3m.py "<map.h3m>" debug
```

Если нужна команда, которой еще нет в `.\analyze_h3m.py`, добавь ее в основной анализатор.

Правила ответов по геймплею:

- Сначала давай практические координаты в формате `(x, y, z)`.
- Предпочитай активную клетку объекта, а не клетку размещения.
- Для вопросов о заклинаниях проверяй святилища, свитки, Хижины Провидца, события, ящики Пандоры и принудительно заданные заклинания в гильдиях магов городов.
- Для вопросов об артефактах проверяй размещенные артефакты, ящики Пандоры, события, Хижины Провидца, монстров, банки существ и артефакты у героев, если режим парсера это поддерживает.
- Если артефакт находится у героя на карте, отвечай, что его можно получить, победив этого героя, и указывай координату героя/объекта.
- Для вопросов о героях проверяй героев на карте, тюрьмы, случайных героев и лагеря героев; класс 62 может быть как Prison, так и Hero Camp.
- Если источник заклинания находится за односторонними монолитами, объясняй маршрут как `entrance -> exit -> nearby source`.
- Не предполагай, что монолит сам дает заклинание; он может только открывать доступ к острову, где находится источник.
- Упоминай, что координаты являются нулевыми координатами парсера/H3M.
